## One-paragraph overview

This project asks: when and why does reading words in context (transformer models) materially improve text classification compared to classical bag-of-words methods? Using DBpedia 14 (~560k Wikipedia article abstracts) we run a three-stage investigation: RQ1 — TF‑IDF + Logistic Regression (baseline); RQ2 — TF‑IDF + Truncated SVD (compression test); RQ3 — DistilBERT fine‑tuning (sample-efficiency 8K vs full-data ceiling). The notebook contains EDA, experiments, explainability (LIME), and deployment guidance suitable for Colab and local runs.

---

## Research questions (explicit)

1. RQ1 — How well does TF‑IDF + Logistic Regression perform on DBpedia 14?
2. RQ2 — Does compressing TF‑IDF features with TruncatedSVD help or hurt performance?
3. RQ3 — Does a contextual model (DistilBERT) outperform TF‑IDF, and how many labeled examples are needed to match or exceed the baseline? (8K vs full)

## Results summary (headline)

- TF‑IDF + Logistic Regression (RQ1): strong baseline (≈98.40% accuracy / Macro F1 on DBpedia). Fast and CPU-friendly.
- TF‑IDF + TruncatedSVD (RQ2): compression hurts performance (≈96.5%); the negative result is instructive — the problem is context-blindness, not overfitting.
- DistilBERT (RQ3): sample-efficient — fine‑tuned on ~8K examples it matches/exceeds TF‑IDF; full-data fine-tune yields marginally higher ceiling performance at much higher compute cost.

See `main_notebook.ipynb` for full tables, confusion matrices, and per-class analysis.
